## Shell-rc

- `.zshrc`: `eval "$(direnv hook zsh)"` + prepend `$HOME/.nix-profile/bin` to PATH.
- `.zprofile`: prepend `$HOME/.nix-profile/bin` to PATH (login shells).
- `~/.config/direnv/direnvrc`: `source "$HOME/.nix-profile/share/nix-direnv/direnvrc"`.

## Per-repo

- Every afc-work repo carries a committed `.envrc` (`use flake quartermaster#<shell>`);
  `direnv allow` on each present repo creates its `.direnv` GC root.

## Cache trust (already in place on the Mini)

- `substrate-1` public key trusted via `/etc/nix/nix.custom.conf`
  (`extra-trusted-public-keys`). The MBP builder signs overlay-built closures
  with the private half (`~/.config/nix/substrate-1.key`) so they arrive
  without rebuild. No new sudo/key moment needed for par.

## Explicitly NOT part of par (machine-specific)

- The private signing key (Mini is a receiver).
- The MBP's ~89-repo working tree (clone only what the Mini runs).
- LaunchAgents / services: Marginalia :3100/:3101, worklog-server, mysql
  datadir, bosun daemons, Atlantis rig.
- MBP-history artifacts: `~/.nvm`, `~/.npm-global`, brew culls, rustup state.
- pantry cache (`~/.cache/ps-pantry/`) — regenerable build accel, not floor.

## Phase-2 note (Nyx capstone)

The devShells + `packages` + `tools` buildEnv in `flake.nix` are the
hand-written Nix a PureScript manifest would author. A manifest like
`toolchain :: Host -> { shells :: Map String (Array Pkg), pins :: Array Pkg }`
compiled via Nyx should regenerate these attrs; conformance = the emitted
Nix matches (or nix-evals equal to) the current hand-written flake outputs.
At 2-host scale there is no per-host branching yet — the value is the
dogfood and the fleet-scale readiness, not present necessity.
